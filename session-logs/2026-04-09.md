    - Pool: `pool`, `assigned_team`
  - **Trip table extensions** (`src/lib/db.ts`): Added 8 columns for mission profiling: `authorized_driver_verified`, `approved_drivers`, `loadout_manifest`, `expected_return_at`, `mission_priority`, `approval_status`, `approved_by`, `am_allocation_ids`
  - **5 new tables** (`src/lib/db.ts`): `driver_vehicle_checks` (full per-spec DDL with 50+ columns), `vehicle_requests`, `scheduled_maintenance`, `post_deployment_checks`, `pr_cost_cache` — all with appropriate indexes
  - **TypeScript types** (`src/types/index.ts`): Added `Vehicle` enrichment fields, `Trip` extensions, and 8 new interfaces: `DriverVehicleCheck`, `VehicleRequest`, `ScheduledMaintenance`, `PostDeploymentCheck`, `PrCostCacheEntry` plus supporting enums/constants
  - **Vehicle API** (`src/app/api/vehicles/route.ts`, `src/app/api/vehicles/[id]/route.ts`): Updated POST to accept all new fields; updated PATCH allowedFields map with 24 new entries
  - **Firestore sync service** (`src/lib/firestore-sync.ts`): Created READ-ONLY sync scaffold with 3 functions:
    - `syncSitesFromFirestore()` — reads `referenceData_sites`, upserts to local `reference_data`
    - `cachePRStatusForWorkOrder()` — reads `purchaseRequests` by PR number, caches to `pr_cost_cache`
    - `fetchAMAllocations()` — reads `am_core_allocations` for in-memory display (no local cache)
    - All functions enforce strict read-only access, transaction rollback on failure, and graceful degradation when Firestore unavailable
